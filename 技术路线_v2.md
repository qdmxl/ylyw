### 为什么这样调整？

- **阶段一已经是抓取**，论文3/4的框架是通用具身行动，所以第二阶段不是重做抓取，而是**在抓取输出上加物理约束层**——这个过渡最自然、改动最小

- **阶段四验证泛化能力**，这是论文4的核心卖点，放在最后独立完成

## 2. 阶段二：物理约束层（当前重点）

### 目标
在现有符号推理输出（抓取策略 → 力预设）的基础上，添加物理定律硬约束，保证输出动作100%物理可行。

### 具体工作

#### 2.1 硬性映射模块 `prior_manual/physics_constraint.py`

```python
class PhysicsConstraint:
    """
    接收顶层推理输出的"意图"（期望力 f_des, 期望力矩 τ_des），
    通过物理定律转化为物理可行的实际指令。
    
    三种模式：
    - hard: 逆动力学解算，100%合规保证
    - soft: 物理损失函数作为惩罚项
    - structural: LNN/HNN（阶段三引入）
    """
```

**核心功能：**
- 力范围裁剪：根据物体质量 m、摩擦系数 μ，计算最小/最大安全抓取力
- 力矩约束：根据关节限位、电机最大扭矩裁剪输出
- 接触力合规：确保法向力 ≥ 0（不穿透）、摩擦力 ≤ μN

#### 2.2 零物理违规验证

在50物体基线测试中增加物理违规计数：
- 穿透检测：抓取力方向是否指向物体内部
- 滑脱检测：摩擦力是否足够克服重力
- 过载检测：关节力矩是否超出电机限制

**预期结果：** 带物理约束的YLYW → 物理违规次数 = 0

#### 2.3 与易理探讨3的对应关系

```
易理探讨3 §2.2 底层                 现有代码改造
─────────────────────────────       ──────────────────
硬性映射（逆动力学）          →     physics_constraint.py (hard模式)
软性引导（物理损失函数）      →     physics_constraint.py (soft模式)  
结构融合（LNN/HNN）           →     留到阶段三
```

### 改动量评估
- 新增文件：`prior_manual/physics_constraint.py`（~300行）
- 修改文件：`prior_manual/prior_manual.py`（在 `get_grasp_strategy` 后增加物理验证调用，~20行）

## 3. 阶段三：通用行动扩展

### 目标
将系统从"只能抓取"扩展到"能推、能倒、能端"的通用行动，注入动态易理规则。

### 3.1 从抓取策略到行动原语

**现有：** 卦象 → 抓取策略（type, force, angle, speed）
**扩展为：** 卦象 → 行动原语（action_type, target_force, target_velocity, constraints）

| 行动类型 | 卦象示例 | 物理参数 | 对应论文 |
|---------|---------|---------|---------|
| grasp | 乾/坤/震/艮/离/坎/兑/巽 | 力 + 角 + 速 | 已有 |
| push | 泰/渐/升/随 | 推力方向 + 速度曲线 | 探讨3 §3 |
| pour | 井/鼎/涣 | 倾斜角 + 角速度 | 探讨4 §3.2 |
| carry | 谦/恒/既济 | 保持力 + 稳定裕度 | 探讨4 §3.3 |

**实现方式：** 扩展 `hexagram_rules.py`，将 `grasp_strategy` 字段泛化为 `action_primitive`，支持多种行动类型。

### 3.2 动态易理规则引擎

这是易理探讨3的核心创新——不只是静态卦象匹配，而是运行时动态触发规则。

新增 `prior_manual/yi_li_rules.py`：

```python
class YiLiRuleEngine:
    """
    易理动态规则引擎
    
    将《易经》核心思想形式化为可触发的模糊规则：
    - 物极必反：状态隶属度 > 阈值 → 反向调节
    - 中正原则：偏离平衡态 → 恢复力
    - 顺承原则：接触力过大 → 柔化退让
    - 否极泰来：连续失败 → 策略切换
    """
```

**规则示例（对应探讨3 §2.1）：**

```
R1 (物极必反):
  IF 倾角_将倒隶属度 > 0.9
  THEN 推力强度 := min(推力强度, 0.2) AND 方向微调(±5°)

R2 (顺承原则):
  IF 接触力 > 物体重量 * 1.5
  THEN 转为顺应模式 AND 跟随物体自然运动趋势

R3 (中正原则):
  IF |当前卦象隶属度 - 既济卦隶属度| > 0.3
  THEN 调节行动参数使状态向既济卦靠拢
```

### 3.3 MuJoCo仿真集成（可选）

在PyBullet中搭建推物场景，或迁移到MuJoCo（论文中提及）。

**最小方案（推荐）：** 在PyBullet中搭建推物/倒水场景，不换仿真器。
- PyBullet已集成，迁移成本为零
- 推物任务：二连杆臂 + 方块 + 目标位置

## 4. 阶段四：分层泛化验证

### 目标
这是易理探讨4的核心贡献——系统性地验证三层泛化能力。

### 4.1 实验一：物理规律泛化

| 实验设置 | 预期 |
|---------|------|
| 训练：推1kg木块 A→B | |
| 测试：推5kg铁球 A→B（零样本） | YLYW 自动适配，MLP 失败 |
| 测试：推0.2kg塑料块 A→B | YLYW 自动适配，MLP 失配 |

**原理：** 物理约束层内置 F=ma，自动按质量缩放力输出。

### 4.2 实验二：结构泛化（核心亮点）

| 实验设置 | 预期 |

**原理：** 推物中学到的"不稳时减速"（物极必反）→ 倒水时"水满时减速"——两者在卦象层面是同一情境。

## 6. 关键技术决策

### 6.1 保持 PyBullet，不迁移 MuJoCo

- PyBullet 已集成，迁移成本高且无必要
- MuJoCo 需要额外安装和环境配置
- 论文中提及 MuJoCo 仅为示例，成果不依赖特定仿真器

### 6.2 阶段二优先用硬性映射，软性引导/结构融合延后

- 硬性映射实现最简单，效果最直接（100%合规保证）
- 软性引导需要训练循环支持，可留到阶段三
- 结构融合（LNN/HNN）属于更前沿的方向，可作为未来工作

### 6.3 64卦补全延后

- 当前22卦已覆盖抓取核心场景
- 扩展到推/倒/端时可逐步补充关联卦象
- 全64卦补全（特别是中孚、小过等抽象卦的工程转译）可作为长期工作

### 6.4 动态规则引擎采用"硬编码+可调参数"混合

- 规则本身硬编码（保持可解释性）
- 触发阈值、调节增益等参数从小样本中学习（如探讨3 R1中的 0.9 阈值）
