**闭环数据流（MuJoCo版）**：

```
MuJoCo物理状态（躯干姿态/位置/速度/关节角）
  → 6D状态向量提取 (姿态→posture, COM高度→com_h, ...)
  → YLYW三层推理 (10Hz)
     L1: 八卦隶属度 → 8卦模糊分类
     L2: 六爻编码 → 6维阴阳判定
     L3: 64卦匹配 → 步态类型+力系数
     L3+: 爻位关系 → 力修正
  → 步态→关节角度目标（髋/膝/踝正弦驱动）
  → MuJoCo执行器力矩输出
  → 物理仿真推进 → 新的状态
```

**MuJoCo vs PyBullet 优势**：
- 真实物理引擎（接触力/摩擦/碰撞检测）
- 可模拟摔倒（躯干倾角过大→地面碰撞）
- 关节力矩直接控制（非运动学动画）
- 更精确的能量/稳定性评估

## Phase 1: MuJoCo 物理闭环验证（本机）

### 目标
在 MuJoCo 物理引擎中搭建完整的闭环仿真：读取机器人真实状态 → YLYW推理 → 关节力矩控制 → 物理推进。
利用真实物理引擎验证摔倒检测、步态切换鲁棒性、自适应修正。

### 当前状态
- `launch_mujoco.py` 已实现开环演示（10种步态，预设状态序列→YLYW→关节动画）
- **需要升级**：开环→闭环（读取MuJoCo状态→YLYW推理→力矩控制）

### 实验1.1：闭环基本步态推理

**目的**：从MuJoCo读取机器人真实状态，YLYW推理步态，力矩驱动关节

**方法**：
- 基于 `launch_mujoco.py` 升级为闭环版本 `launch_mujoco_closed_loop.py`
- 从 `data.qpos` / `data.qvel` 提取6D状态（躯干姿态、COM高度、力分布、ZMP、扰动、地形）
- YLYW推理→步态参数→髋/膝/踝正弦目标角度→位置控制力矩
- 步态切换由YLYW自动完成（不再预设场景序列）

**6D状态提取映射**：
| 维度 | MuJoCo数据源 | 归一化方式 |
|---|---|---|
| posture | 躯干倾角 (roll+pitch) | 1 - tilt/0.5 |
| com_h | 躯干Z位置 | COM_z / 1.1 |
| force_dist | 左右脚接触力比值 | 1 - |F_left - F_right|/F_total |
| zmp_margin | COM投影到支撑多边形 | 距边界距离/半宽 |
| disturbance | 躯干角速度幅度 | min(1, ang_mag/3) |
| terrain | 地面法向量（或默认） | 0.8 平坦 |

**指标**：与开环版本相同的11种场景，对比推理一致性

---

### 实验1.2：物理摔倒检测与恢复

**目的**：利用MuJoCo真实物理，验证YLYW在摔倒前后的推理切换

**方法**：
- YLYW输出过高速步态 + 地面摩擦力降低 → MuJoCo物理模拟真实摔倒
- 躯干触地（Z < 0.4m）→ 检测摔倒 → YLYW应自动推理出恢复卦
- 记录摔倒前→摔倒→恢复的卦象切换路径

**关键观察**：
```
奔跑(大壮卦, run) → 高速+冰面 → 躯干倾角增大 → 
YLYW感知: posture↓ zmp↓ disturbance↑ → L2编码变化 → 
匹配变化: 大壮→坎→复→渐 → 步态切换: run→recovery→slow_walk
```

### 实验1.3：自适应学习（MuJoCo闭环）🆕

**目的**：在MuJoCo物理闭环中集成 `YLYWAdaptiveController`，验证自适应修正

**方法**：
- 替换 `YLYWLocomotionController` 为 `YLYWAdaptiveController`
- 反馈信号直接从MuJoCo物理状态提取（非模拟）：
  - `fell`: 躯干Z < 0.35m
  - `com_deviation`: |COM_Z - 0.95| 
  - `zmp_margin`: COM投影距支撑边界
  - `speed_error`: 实际滑动速度 vs 期望速度

**实验场景**：
| 场景 | 注入异常 | 预期自适应行为 |
|---|---|---|
| 3.1 髋关节偏置 | 左髋零位+5° | YLYW检测COM偏移→修正COM爻阈值 |
| 3.2 地面摩擦突变 | 摩擦系数 1.0→0.3 | 检测滑动→匹配谨慎卦→降低力系数 |
| 3.3 单腿电机弱化 | 右膝力矩×0.6 | 力分布不均→爻位关系修正→降低整体力系数 |
| 3.4 初始模板污染 | 随机偏移5个卦模板 | GT反馈→逐步恢复→模板偏差趋势图 |

**指标**：
| 指标 | 静态YLYW | 自适应YLYW |
|---|---|---|
| 异常后摔倒次数 | 基线 | 降低>50% |
| COM恢复时间 | 基线 | 缩短>30% |
| 持续异常的步态稳定性 | 持续恶化 | 自适应收敛 |

## Phase 2: 灵犀X2实物部署

### 目标
将 YLYW 推理引擎部署到灵犀X2机载计算平台，验证真实传感器→YLYW→运动控制闭环。

### 准备条件

- [ ] 灵犀X2机器人（ROS2 Humble + aimdk_msgs）
- [ ] 平坦测试场地（至少 5m × 5m）
- [ ] 安全绳/保护垫
- [ ] 录屏设备（记录终端输出 + 机器人动作）

### 实验2.1：静止站立 → 慢走 → 行走 → 慢走 → 站立

**目的**：最基本的步态切换序列，验证 YLYW 在实物上的推理稳定性

**步骤**：
1. 启动 ROS2 节点 `python3 ylyw_lingxi_locomotion.py`
2. 先静止站立 5s（采集 IMU 基线）
3. 切换到慢走 10s
4. 切换到行走 10s
5. 切换到慢走 10s
6. 切换回站立 5s
7. Ctrl+C 安全退出

**数据**：
- 终端日志：每5s输出 卦象+步态+速度+tilt
- 手动记录：步态切换是否平滑、有无异常抖晃
- IMU数据：可 ros2 bag record 保存

**安全检查单**：
- [ ] 倾角>23°时自动降速（tilt > 0.4rad）
- [ ] 倾角>40°时紧急停止（tilt > 0.7rad）
- [ ] 手动Ctrl+C后零速退出
- [ ] 无IMU数据时零速保护

---

### 实验2.2：全步态覆盖测试

**目的**：覆盖所有11种步态类型，验证速度映射表的合理性

**序列**：
```
站立(5s) → 慢走(8s) → 谨慎行走(8s) → 行走(8s) → 快速行走(8s)
  → 小跑(5s) → 奔跑(3s) → 紧急停止 → 恢复步态(5s) → 自适应(8s) → 站立(5s)
```

**安全策略**：
- 奔跑不超过3s
- 小跑/奔跑前确认场地无遮挡
- 安全绳待命

**指标**：
| 指标 | 记录方式 |
|---|---|
| 每个步态的实际速度 | 从 odometry 或手动计时 |
| 步态间切换是否产生震荡 | 肉眼观察 + IMU数据 |
| tilt 最大值 | 终端日志 |
| 卦象切换路径 | 终端日志 |

---

### 实验2.3：重复性测试

**目的**：验证 YLYW 在相同条件下输出的一致性

**方法**：
- 执行 实验2.1 三遍
- 对比三遍的卦象序列、步态序列、速度序列

**指标**：
- 卦象序列一致性：三遍相同的步态→卦象映射是否一致
- 速度序列相关系数：> 0.9

## Phase 3: 自适应学习（实物+仿真混合）

### 目标
在灵犀X2实物上实现 YLYW 自适应学习的完整闭环，
这是论文的核心亮点。

### 实验3.1：传感器偏置模拟 → 自适应恢复（仿真+实物）

**目的**：故意引入传感器偏置，观测 YLYW 自适应修正

**仿真验证（先）**：
- 修改 IMU 模拟器，对加速度Z轴加 +0.5m/s² 偏置
- 这会导致 COM高度编码偏高 → 匹配到更高速的卦 → 实际速度过快
- 自适应控制器应检测到速度偏差 → 修正 COM高度爻的阈值

**实物验证（后，需谨慎）**：
- 在 ROS2 节点的 IMU→状态映射器中，对 Z 加速度加一个小偏置（0.2m/s²）
- 观测自适应修正是否触发

**指标**：
| 指标 | 静态YLYW | 自适应YLYW |
|---|---|---|
| 偏置注入后速度偏差 | 持续存在 | 5步内修正 |
| 修正后速度恢复 | 不会恢复 | 恢复到偏置前水平 |
| 修正记录可读性 | N/A | 完整推理链 |

---

### 实验3.2：负载变化自适应

**目的**：灵犀X2加载额外重量（如背包），YLYW 自动调整力系数

**方法**：
1. 先无负载行走 20s，记录力系数基准
2. 加 2kg 负重
3. 继续行走 20s，观测力系数是否自动上调
4. 对比有无自适应的区别

**预期**：
- 静态 YLYW：负重后步态不变 → 可能发生倾斜/减速
- 自适应 YLYW：负重后力系数自动上调 → 维持步态稳定性

---

### 实验3.3：长期漂移补偿

**目的**：长时间运行后，IMU 零点漂移 → YLYW 自动校准

**方法**：
- 灵犀X2持续慢走 5 分钟
- 记录每30秒的 L2阈值、模板偏差、修正次数
- 观测是否有稳定的自校准趋势

**指标**：
- 漂移量与修正量的相关性
- 5分钟内的修正总次数和分布
- 修正后的稳定性是否改善

## Phase 4: 论文级实验

### 目标
产出可发表的实验数据和图表

### 实验4.1：消融实验（方法论验证）

| 消融项 | 说明 | 预期影响 |
|---|---|---|
| **完整 YLYW** | L1+L2+L3+爻位关系 | 最佳性能 |
| **-L1（无八卦）** | 6D状态直接进L2编码 | 类别区分度下降 |
| **-爻位关系** | 去除力修正系数 | 扰动恢复能力下降 |
| **-64卦（8卦）** | 仅用八卦直出步态 | 步态粒度不足 |

每个消融变体在仿真中运行全部11种场景100次，记录：
- 步态合理率
- 扰动恢复时间
- 摔倒频率

### 实验4.2：对比基线

| 基线 | 实现方式 | 比较维度 |
|---|---|---|
| **固定步态表** | 状态阈值→步态查表（不含易经符号） | 鲁棒性 |
| **模糊逻辑** | 状态→模糊规则→步态（无卦象层） | 可解释性 |
| **MLP** | 6D→64D→步态分类（小网络） | 泛化能力 |
| **PID 速度控制** | 目标速度→PID→关节力矩（灵犀MC默认） | 场景适应性 |

### 实验4.3：跨域泛化

验证 YLYW 从仿真→实物的零样本迁移能力：
- 仿真中训练/调整的参数，直接用于实物
- 不需要实物微调（零样本）
- 对比 MLP 基线（需要实物微调）

### 实验4.4：统计显著性

- 每个实验至少 5 次独立重复
- 报告 mean ± std
- 成对 t-test 或 Wilcoxon signed-rank test
