## 阶段二：物理约束层 → 双八卦安全架构 ✅ 已完成

**时间：2026年6月2-3日**

### 架构演进

原计划使用逆动力学硬性映射 + 力/力矩合规验证实现物理约束层。实际实现中提出了更优雅的方案：**双八卦安全架构**——将物理约束完全符号化为第二个六十四卦系统（安全八卦），与策略八卦并行运行，通过仲裁层合并输出。

### 核心设计

```
13维特征 ──┬──▶ 策略八卦（prior_manual）→ 策略类型 + 力预设
           │
           └──▶ 安全八卦（safety_bagua） → 安全等级 + 修正系数
                    │
                    ▼
              双八卦仲裁层 ──▶ 最终安全指令
```

**安全八卦**共享与策略八卦完全相同的三层架构（L1八卦→L2六爻→L3六十四卦），但语义不同：
- L1: 8卦对应8种物理安全原型（稳定/脆弱/光滑/超重等）
- L2: 6爻对应6条物理安全检查公式（摩擦力/破坏阈值/摩擦锥/关节力矩/力封闭/穿透风险）
- L3: 64卦映射为5级安全等级（SAFE/CAUTION/WARNING/DANGER/CRITICAL）

### 安全等级与修正

| 安全等级 | 力修正 | 速度 | 策略 |
|---------|:---:|------|------|
| SAFE | ×1.00 | 原速 | 放行 |
| CAUTION | ×0.95 | slow | 放行 |
| WARNING | ×0.80 | slow | 放行 |
| DANGER | ×0.60 | slow | cautious族 |
| CRITICAL | ×0.30 | slow | 触发变卦 |

### 实验数据

304物体物理合规基线测试：SAFE 38.8%、CAUTION 0.3%、WARNING ~52%、DANGER ~4%、CRITICAL ~0.3%。脆弱物体（花瓶、灯泡）力修正×0.60~0.80，极端场景触发完整五级安全等级。

### 关键创新

1. **符号化物理约束**：6条物理公式仅用于L2层的阴阳二值化，L3后完全在符号空间推理，零数值计算
2. **双八卦并行**：两个64卦系统独立运行，共享架构但语义互补
3. **变卦触发**：CRITICAL级别时安全八卦的违规状态映射回策略八卦，请求重新选卦
4. **全可解释**：每个安全决策可追溯到具体爻位的物理违规

### 产出

- `safety_bagua/__init__.py` — 包定义
- `safety_bagua/safety_yao_encoder.py` — 安全六爻编码器
- `safety_bagua/safety_hexagram_rules.py` — 安全64卦规则
- `safety_bagua/dual_bagua_arbiter.py` — 双八卦仲裁器
- `safety_bagua/paper_v05_draft.md` — 论文§5草案

## 视觉分类分支 ✅ 已完成

**时间：2026年6月5日**

### 核心思想

验证易经符号先验的跨域泛化能力：将物理域的八卦原型替换为视觉域的八卦原型，复用八卦-六十四卦推理架构，构建零样本图像分类器。

### 技术方案

采用"一卦一算子"设计：8个卦象各对应一个专用视觉算子（角点规整度/局部平滑度/Gabor方向主导度/GLCM对比度/梯度方向熵/亮度峰值密度/大块同质占比/高光局部对比），构成8维特征向量，经Z-score归一化后以欧氏距离最近原型分类。

### 实验结果

| 数据集 | 类型 | Top-1 | Top-3 | 说明 |
|--------|------|:---:|:---:|------|
| 合成纹理 | 纹理 | 100% | 100% | 原型完美匹配 |
| 合成自然物体 | 物体 | 89.6% | 99.3% | 5折交叉验证 |
| Brodatz | 真实纹理 | 14.0% | 55.2% | 类间模糊 |
| **STL-10** | **真实物体** | **37.0%** | **75.0%** | **标准基准, 3×随机** |

### 爻位关系验证

在静态图像分类中，爻位关系（乘承比应当位得中）修正系数0.972±0.059，接近恒等映射。**验证了爻位关系设计用于处理动态变化中的因果协调，而非静态判别**——为时序视觉和触觉感知提供了明确的架构定位。

- 特征提取：8D简单算子(simple_8d.py)、52D丰富特征(rich_features.py)、专用检测器
- 测试脚本：STL-10、Brodatz、合成物体、小样本微调、爻位关系分析

## 芯片化路线 🆕 远期规划

**提出时间：2026年6月5日**

### 核心理念

YLYW的推理计算以比较、查表、计数为主，天然适合纯逻辑电路实现。与深度学习芯片（NPU/TPU）依赖乘加阵列（MAC）不同，YLYW芯片可用比较器+加法器+查找表实现，无需DRAM带宽瓶颈。

### 芯片化优势

### 推理链路的门级映射

```
L1 八卦隶属度（8路并行）:
    欧氏距离 = (x-p)² 求和 → 减法器 + 平方器 + 加法器
    exp查找 → ROM查找表（256×8bit）

L2 六爻编码（6路并行）:
    线性组合 = Σw_ij × f_j → 固定系数乘法（移位加）+ 加法器

L3 64卦匹配:
    余弦相似度 = 点积 → 并行比较器阵列 → 最大值选择器

L3+ 爻位关系:
    当位: 比较器（爻值 vs 0.5）→ 计数器
    得中: 二爻/五爻专用比较器
    乘承: 相邻爻比较器阵列 → 计数器
    比: 相邻爻异或门 → 计数器
    应: 初↔四/二↔五/三↔上异或门 → 计数器
    综合评分: 固定系数加权求和器
```

### 量子YLYW的核心优势

**量子并行推理**：经典YLYW每次只能查询一个卦（一个确定状态），量子YLYW可同时处于64卦的叠加态：

```
经典: 输入→匹配→选1卦→1种策略  O(64)
量子: 输入→|ψ⟩=Σc_i|H_i⟩→64卦同时推理→干涉选出最优→测出结果  O(1)
```

这不是"算得更快"的问题——量子YLYW可以在64卦的指数级叠加空间中做经典计算不可能实现的并行推理。

**量子爻位关系**：在经典域，乘承比应当位得中是6次独立的比较/计数操作。在量子域，它们是作用在6-qubit态上的纠缠门序列，一次执行即可完成全部关系分析。

**量子类比推理**：两个物体的量子态|ψ_A⟩和|ψ_B⟩的保真度F=|⟨ψ_A|ψ_B⟩|²天然度量"相似性"，可在64卦希尔伯特空间中做类比推理——这是经典计算无法高效实现的。

### 核心理念

YLYW 是可解释推理系统，每一步推理都是人类可读的。这使得自适应学习可以走一条与黑箱RL完全不同的路线：一次失败，追溯推理链，定位责任层（L1/L2/L3/爻位关系），修正对应参数（仅~443个），下一次就成功。

```
失败时：
  DRL/端到端 → "某个隐藏层权重需要调整"（不可审计，需大量样本）
  YLYW     → "第3爻（摩擦力）被判为阳，实际应阴，导致匹配大壮卦"（可精确定位，一次修正）
```

```

  │                                                ↓
  │                                          观测结果O
  │                                                ↓
```

### 技术架构

```
观帖（视觉YLYW）
  ├─ 楷体字帖 → 12维笔画特征提取
  ├─ 每笔画八卦隶属度计算（类型+方向+尺度+位置）
  ├─ 笔画间爻位关系分析（乘/承/比/应/接/齐）
  └─ 六十四卦全局结构匹配
        ↓
规划（规则库）
  ├─ 64卦规则库（上下卦插值生成）
  ├─ 每笔画笔法选择（乾→中锋/坤→侧锋/震→提按...）
  └─ 爻位关系修正执行参数（乘+起笔压力/接-起笔压力...）
        ↓
执行（MuJoCo仿真）
  ├─ 人工标注笔画端点→二次贝塞尔曲线轨迹
  ├─ 规则库指令控制压力曲线（起笔顿→行笔提→收笔回锋/出锋）
  └─ thickness_ratio直接控制笔触渲染半径
        ↓

  ├─ 全局IoU + 覆盖率

        ↓
精进（知几学习）
  ├─ 爻位偏差→相对位置修正
  ├─ 覆盖率不足→thickness_ratio微调（上限1.2防粘连）
  └─ IoU低→减细+微调
```

### 实验结果

| 汉字 | 笔画数 | 初始分 | 最终分 | 改善 | 迭代轮数 | 关键发现 |

1. **爻位关系诊断有效**：系统能正确检测"撇↔捺爻位偏差0.991"，定位问题笔画对

4. **自动笔画分割受限**：Zhang-Suen骨架在粗笔画楷体字帖上产生毛刺→改用人工标注
5. **覆盖与分离的平衡**：thickness_ratio上限1.2是保证笔画不粘连的临界值

## 知几学习：从"事后奖惩"到"事先征兆"的学习范式革新 🆕

**提出时间：2026年6月9日（易理探讨9）**

### 核心思想

将《易经》"知几"哲学工程化为一种全新的具身学习范式。所谓"几"，即《系辞传》所言："几者，动之微，吉之先见者也。君子见几而作，不俟终日。"——事物变化最微小的征兆。知几学习的内核是：**真正的智慧不在于事后奖惩，而在于事先洞察变化的征兆，在正确时机执行合规行动。**

### 与强化学习的根本区别

| 维度 | 强化学习（RL） | 知几学习（ZJ-Learning） |
|------|--------------|----------------------|
| 时间性 | 事后驱动：奖励在行动之后 | 事先驱动：征兆刚出现就响应 |
| 驱动力 | 外部评价：环境奖励信号 | 内部评价：内建吉凶价值体系 |
| 知识起点 | 白板，从零探索 | 内建先验，具备变化哲学与物理常识 |
| 探索方式 | 盲目随机，全局试错 | 启发式，先验范围内局部校准 |
| 学习对象 | 从零学习策略映射 π(a|s) | 仅校准征兆辨识的临界阈值 |
| 样本复杂度 | O(ε^(-N))，N维状态空间 | O(ε^(-K))，K个可调参数（K≪N） |
| 安全性 | 依赖数据\"喂\"出安全行为 | 结构内建，天生物理合规 |
| 可解释性 | 黑箱 | 透明推理链，每步可追溯 |

### 数学形式化

**征兆空间 O：** 模糊征兆隶属度函数集合，由人类先验知识预定义。每个元素 o_i 代表一种变化征兆的隶属度（如"将倾"、"将稳"、"将碰"的临界状态）。

**知几策略：** π_ZJ: S →(fuzzy) O →(rules) A，模糊化层基于先验隶属度函数推理，规则层基于先验规则库决策。两部分在零样本下即可工作。

**知几学习目标：** min_θ L_calibration(θ) + λ·L_safety(θ)，校准误差使"几"的判断更精准，安全约束维护物理合规边界。没有外部奖励信号。

**知识来源的根本差异：** Knowledge_RL = f(Data)，而 Knowledge_ZJ = Prior ⊕ f_calibration(Data)。知几学习的大部分知识来自先验而非数据。

### 与YLYW架构的关系

知几学习是YLYW三层推理架构天然对应的学习范式：
- **L1 八卦隶属度**：征兆模糊化层的可计算实现
- **L2 六爻编码**：征兆聚合为结构化状态
- **L3 卦象匹配+爻位关系**：先验规则推理层

YLYW的单智能体验证（物理域92.7%、运动控制域14种步态、视觉分类域37.0% Top-1）已经提供了知几学习"零样本即可合理"的实证基础。层次化嵌套YLYW架构则为知几学习提供了"从单智能体到分布式智能生态"的扩展路径。

### 与现有路线的衔接

| 现有路线 | 知几学习视角下的重新定位 |
|---------|----------------------|
| 阶段一（符号先验推理） | 知几学习的零样本基线证明 |
| 阶段二（双八卦安全架构） | 知几学习的安全约束形式化 |

> **"智能体的学习，不必从一张白纸开始。"** 知几学习不是对强化学习的否定，而是一种必要的补充和超越——强化学习负责在未知情境中探索创新（系统2），知几学习负责日常的、物理合规的快速直觉反应（系统1）。两者形成的混合架构，为通往更高效、更安全、更可解释的具身智能提供了全新的理论基础。

## 宏观-微观生成器架构路线 🆕 关键突破

**提出时间：2026年6月21日**

### 核心思想

ALFWorld、书法、步态控制三个实验的比较揭示了同一个瓶颈：**YLYW的宏观符号推理与微观精确控制之间存在鸿沟**。但步态控制的成功（G1→CPG正弦波生成关节轨迹）和ALFWorld的成功（环境→提供完美Simulator）都指向了同一个解决方案——**在宏观YLYW和微观执行之间插入一个生成器层**。

更重要的是，这个生成器本身可以是**固定规则**（如CPG），也可以是**微型YLYW子模型**——这正是层次嵌套架构在单智能体层面的直接应用。

### 三层生成器架构

```
┌─────────────────────────────────────────────────────────┐
│  宏观YLYW（父模型 — L3六十四卦规则库）                   │
│  输入：任务级感知（6维状态向量 / 12维笔画特征）          │
│  输出：卦象意图（步态类型 / 笔法选择 + 宏观参数）        │
│  角色："做什么" — Planner                               │
├─────────────────────────────────────────────────────────┤
│  生成器层（固定规则 或 微型YLYW子模型）                  │
│  输入：父模型的卦象意图 + 当前执行状态                   │
│  输出：精确的执行参数（关节角度 / 轨迹坐标 + 压力）      │
│  角色："怎么做" — Simulator → Actuator 的桥梁            │
│                                                         │
│  固定生成器：CPG正弦波、Catmull-Rom样条、贝塞尔曲线      │
│           优点：简单可靠，确定性强                       │
│           局限：只能处理规律性任务                       │
│                                                         │
│  微型YLYW子模型：每个关节/笔画点为独立元胞               │
│           优点：自适应、可学习、可解释                   │
│           局限：计算量较大，需要反馈信号                  │
├─────────────────────────────────────────────────────────┤
│  物理执行层（MuJoCo / 真实机器人）                       │
│  输入：生成器输出的精确参数                              │
│  输出：物理运动 + 传感器反馈                             │
│  角色："实际动作" — Actuator                            │
└─────────────────────────────────────────────────────────┘
```

### 微型YLYW子模型的设计

每个子模型是一个极简的YLYW单元：

- **八卦基元**：只用2-3个卦（如坎="接近极限/危险"、艮="在安全范围/保持"、震="需要快速响应"）
- **六爻编码**：只用1-2个爻位（如"当位"表示关节角度是否偏离理想位置、"得中"表示是否在运动范围中点）
- **规则**：3-5条核心模糊规则（如"当位得中则保持"、"偏离则修正"、"接近极限则减速"）
- **学习**：知几校准——只微调临界阈值

子模型的优势在于：
1. **与父模型共享八卦符号语言**：父模型发"震卦"意图，子模型自然理解"需要快速动态响应"
2. **子模型自主决策，父模型不干预微观**：父模型不需要知道28个关节各自的角度，只需要给出意图
3. **子模型之间通过爻位关系协调**：相邻关节的协调可以通过"比"（相邻和谐）关系自然实现

### 在三个域的实例化

| 域 | 宏观YLYW输出 | 固定生成器 | 微型YLYW子模型 |
|------|------------|-----------|---------------|
| **步态控制** | 步态类型+speed/freq/force | CPG正弦波→关节角度 | 每关节检测"当位/得中"→微调角度 |
| **书法** | 笔法类型+压力基准+起收笔 | Catmull-Rom样条→轨迹点 | 每轨迹点检测"当位"（离模板距离）→微调坐标 |
| **ALFWorld** | PDDL目标+阶段意图 | admissible_commands过滤 | （环境已提供完美Simulator，子模型不需要） |

### 验证计划

**步态控制（已有闭环基础，当前即可验证坡度自适应）**：

1. **闭环链路已完整**：`launch_mujoco_closed_loop.py` 实现了 `MuJoCo真实状态 → 6维向量 → YLYW推理 → 卦象+步态 → CPG关节角度 → MuJoCo执行` 的完整闭环
   - `MujocoStateExtractor` 从物理引擎读取躯干倾角、COM高度、关节力矩、ZMP裕度等，转换为6维归一化状态
   - 已支持 terrain 参数（ice: 摩擦0.15 / normal / rough）
   - 已支持外部推搡（`schedule_push`）和摔倒检测+自动恢复
   - **关键：每一步的6维状态是物理引擎的真实值，不是预设值**——当环境变化（坡度、摩擦、推搡），这6维值自然变化，YLYW推理自然输出不同的卦象和步态

2. **坡度自适应无需额外开发**：只需将 floor geom 绕y轴旋转即可模拟坡度。IMU（躯干xquat）检测到 pitch 偏移 → `posture/tilt` 维度变化 → 6维状态送入YLYW → 卦象自动从"离卦（行走）"变为"艮卦（爬坡谨慎）"或"蹇卦（艰难上坡）" → 步态参数自然调整。固定CPG负责把新步态参数转为关节角度。

3. **微型YLYW关节子模型的增量价值**：
   - 当前固定CPG用统一 `hip_amp/knee_amp` 套给所有关节
   - 上坡时踝关节需要更大背屈（维持脚掌着地），冰面时步幅需缩小步频需提高——固定CPG做不到逐关节差异化
   - 关节级微型YLYW：当踝子模型检测到"足底接触力不平衡"（离卦→坎卦）→ 独立增加踝背屈修正，不改变髋膝
   - **预期对比**：坡度突变时，固定CPG需等下一轮YLYW推理（~100ms延迟）才能调整；关节子模型可以在~10ms内通过局部"当位"检测完成微调

**书法（短期，在现有闭环中扩展）**：
1. 将当前的"YLYW→端点→直线样条→轨迹点"改为"YLYW父→笔画CPG子模型→轨迹点"
2. 每个轨迹点子模型输入：当前坐标、目标坐标（父模型意图）、压力——输出：修正后的坐标+压力
3. 预期：子模型的"当位"检测将自动修正笔画偏移，解决当前"爻位诊断但无法精确修正"的问题

### 从固定生成器到微型YLYW的渐进路线

```
第一步：固定生成器验证（1-2周）
  ├─ 书法：Catmull-Rom样条 + 起收笔压力曲线 → 验证"笔画生成器"概念
  ├─ 步态：CPG正弦波 → 验证"步态生成器"概念
  └─ 产出：生成器抽象接口 + 两个域的实例

第二步：微型YLYW子模型（2-4周）
  ├─ 每个轨迹点/关节 → 极简YLYW（2-3八卦 + 1-2爻位 + 3-5规则）
  ├─ 子模型通过"当位/得中"自主修正
  └─ 产出：子模型架构 + 对比实验数据

第三步：父-子协同学习（4-8周）
  ├─ 父模型通过子模型的"卦象反馈"来调整宏观策略
  ├─ 子模型通过知几学习校准自己的阈值
  └─ 产出：完整的双层YLYW学习闭环
```

### 核心主张

> **"YLYW的宏观策略能力是真实的，问题不在于策略错误，而在于缺少将策略转化为精确动作的中间层。这个中间层可以是固定的物理规则（如CPG），也可以是一个遵循同样爻位法则的微型YLYW子模型。两者不是互相排斥的——固定生成器保证基线和安全，微型子模型提供自适应能力。"**
