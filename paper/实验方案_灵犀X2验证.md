## 一、实验目标

在智元灵犀X2双足机器人 + 灵巧手平台上，完成 YLYW 系统从**感知→八卦→六爻→卦象→爻位→决策→执行**的完整闭环物理验证，为论文提供实物实验证据。

### 核心验证点

| 验证点 | 对应论文贡献 | 验收标准 |
|--------|-------------|---------|
| 13维特征在线提取 | L1 八卦基元输入 | 视觉系统稳定输出归一化特征 |
| 64卦策略映射 | L3 六十四卦规则 | 不同物体正确匹配不同卦象 |
| 爻位关系→力修正 | L3+ 爻位关系运算 | 脆弱物体自动降力，重物自动加力 |
| 灵巧手多策略执行 | 决策层输出 | 39种策略中至少验证6-8种 |
| 端到端延迟 | 工程实现 | 感知→执行 < 100ms（不含机械运动） |

## 四、实验设计

### 实验一：零样本基线测试（主实验）

**目的**：复现论文 Table 10 的实物结果，验证纯先验推理的物理可行性。

**步骤**：
1. 8类物体，每类5-10个实例（总计40-80个）
2. 每个物体放在固定位置，视觉系统提取13维特征
3. YLYW 推理输出策略类型 + 力预设 + 接近角度 + 速度等级
4. 灵巧手执行抓取
5. 人工评估：策略合理性（合理/中性/错误）

**记录指标**：
| 指标 | 记录方式 |
|------|---------|
| 策略合理率 | 人工标注 |
| 严重错误率 | 抓取失败/物体损坏 |
| 推理延迟 | Python计时 |
| 执行成功率 | 物体被成功抓取并放置 |
| 匹配卦象 | 每个物体的Top-1卦象 |
| 爻位评分 | S_yao + modifier |
| 力修正系数 | modifier 实际值 |

**预期**：合理率 ≥ 85%（实物环境比仿真噪声大，略低于92.7%是正常的）

### 实验二：爻位关系→力修正验证（特色实验）

**目的**：证明"爻位关系决定执行参数"——脆弱物体自动降力。

**步骤**：
1. 选两组对比物体：
   - **脆弱组**：花瓶（天泽履）、陶瓷碗（坤为地）、玻璃瓶（地火明夷）
   - **坚固组**：木块（乾为天）、石块（火泽睽）、金属罐（雷天大壮）
2. 每组5个实例，共30次实验
3. 记录每次的爻位评分 S_yao 和力修正系数 modifier
4. 对比两组 modifier 均值

**预期**：脆弱组 modifier < 1.0（降力），坚固组 modifier ≥ 1.0

### 实验三：卦象-策略多样性展示（演示实验）

**目的**：直观展示"不同物体→不同卦象→不同策略"。

**步骤**：
1. 每种物体类型选1-2个代表，逐一演示
2. 录屏显示 YLYW 推理过程（隶属度→爻向量→匹配卦象→策略）
3. 录像显示灵巧手实际抓取姿态

**预期成果**：一段 3-5分钟的演示视频（arXiv 可附，极有说服力）

### 实验四：视角鲁棒性（可选）

**目的**：验证感知层对视角变化的鲁棒性。

**步骤**：
1. 同一物体，从3个不同角度采集特征
2. 比较3次推理输出的策略是否一致
3. 记录6维爻向量的方差

**预期**：策略一致性 ≥ 80%

## 五、特征提取方案

### 5.1 13维物理特征及其视觉估算方法

| # | 特征 | 视觉估算方法 | 所需传感器 |
|---|------|-------------|-----------|
| 1 | 稳定性 (stability) | 底面支撑面积 / 物体高度 | RGB-D |
| 2 | 滚动倾向 (roll_tendency) | 曲率半径 / 接触面平坦度 | RGB-D |
| 3 | 力需求 (force_needed) | 物体体积 × 材质密度估计 | RGB-D |
| 4 | 脆弱性 (fragility) | 材质识别（玻璃/陶瓷→高，木材/金属→低） | RGB (+CNN) |
| 5 | 抓取表面质量 (grasp_surface_quality) | 表面纹理粗糙度 | RGB |
| 6 | 支撑面积 (support_area) | 底部轮廓面积 | RGB-D |
| 7 | 遮挡程度 (occlusion) | 深度图完整性 | RGB-D |
| 8 | 可达性 (reachability) | 物体与灵巧手相对位置 | RGB-D |
| 9 | 障碍密度 (obstacle_density) | 工作空间占用率 | RGB-D |
| 10 | 任务优先级 (task_priority) | 人工设定（实验固定为0.5） | — |
| 11 | 重量比 (weight_ratio) | 体积 × 材质密度 / 灵巧手负载 | RGB-D |
| 12 | 可见性 (visibility) | 物体表面可见像素比例 | RGB-D |
| 13 | 变形能力 (deformability) | 材质识别（橡胶/泡沫→高） | RGB |

> **简化方案（Phase 1）**：前8个特征用几何计算，后5个（任务优先级、重量比、可见性、变形能力、障碍密度）用预设常量。等跑通全流程后再替换为真实视觉估计。

### 5.2 视觉处理流水线

```python
# 伪代码：特征提取器接口
class VisualFeatureExtractor:
    def __init__(self, camera_config):
        self.camera = RealSenseCamera(camera_config)
        self.material_classifier = load_material_cnn()  # 可选的材质识别模型
    
    def extract(self, rgb_image, depth_image) -> np.ndarray:
        """输入 RGB + 深度图，输出 13维归一化特征向量 [0,1]¹³"""
        mask = self.segment_object(rgb_image, depth_image)    # 物体分割
        bbox = self.get_bbox(mask)                            # 边框
        point_cloud = self.depth_to_pointcloud(depth_image, mask)  # 点云
        
        # 几何特征（点云计算）
        stability = self.compute_stability(point_cloud)
        roll_tendency = self.compute_roll_tendency(point_cloud)
        force_needed = self.estimate_volume(point_cloud) * self.estimate_density(rgb_image[mask])
        support_area = self.compute_support_area(point_cloud)
        occlusion = 1.0 - (mask.sum() / bbox.area)
        reachability = self.compute_reachability(point_cloud)
        
        # 纹理/材质特征（RGB计算）
        grasp_quality = self.compute_surface_quality(rgb_image[mask])
        fragility = self.material_classifier.predict(rgb_image[mask])
        
        # 固定特征（Phase 1）
        task_priority = 0.5
        weight_ratio = force_needed / MAX_PAYLOAD
        visibility = mask.sum() / (rgb_image.shape[0] * rgb_image.shape[1])
        deformability = self.material_classifier.predict_deformability(rgb_image[mask])
        obstacle_density = 0.1  # 实验台环境
        
        return np.array([stability, roll_tendency, force_needed, fragility,
                        reachability, grasp_quality, support_area, occlusion,
                        obstacle_density, task_priority, weight_ratio,
                        visibility, deformability])
```

### 7.1 主控脚本伪代码

```python
def experiment_zero_shot():
    """实验一：零样本基线测试"""
    camera = VisualFeatureExtractor(config)
    ylyw = YLYWSystem(hexagram_rules='full_64')
    robot = LingxiX2Adapter(sdk_client)
    
    results = []
    for obj_id, obj_type in enumerate(objects):
        # Step 1: 感知
        rgb, depth = camera.capture()
        features = camera.extract(rgb, depth)  # 13维 [0,1]
        
        # Step 2: YLYW 推理
        t0 = time.time()
        trigram_membership = ylyw.L1.compute(features)      # 8维隶属度
        yao_vector = ylyw.L2.encode(features)                # 6维爻向量
        hexagram, similarity, top3 = ylyw.L3.match(yao_vector)  # 卦象匹配
        S_yao, modifier = ylyw.L3plus.analyze(yao_vector)    # 爻位关系
        strategy = ylyw.decision.output(hexagram, S_yao, modifier)
        t_total = time.time() - t0
        
        # Step 3: 执行
        success = robot.execute_strategy(strategy)
        
        # Step 4: 记录
        results.append({
            'obj_id': obj_id,
            'obj_type': obj_type,
            'hexagram': hexagram.name,
            'similarity': similarity,
            'strategy_type': strategy['strategy_type'],
            'force_preset': strategy['force_preset'],
            'modifier': modifier,
            'S_yao': S_yao,
            'inference_ms': t_total * 1000,
            'execution_success': success,
        })
        
        print(f"[{obj_id:3d}] {obj_type:10s} → {hexagram.name:20s} "
              f"→ {strategy['strategy_type']:20s} "
              f"force={strategy['force_preset']*modifier:.2f} "
              f"({t_total*1000:.1f}ms)")
    
    # 汇总
    save_results(results)
    print_summary(results)
```

## 九、风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| 灵巧手力控精度不足 | 无法体现力修正效果 | 用开合度替代力控，二进制对比 |
| 视觉特征提取不准 | 合理率大幅下降 | 先用人工标注特征跑通流程，再迭代视觉 |
| SDK接口限制 | 某些策略无法实现 | 简化为6种核心策略覆盖即可 |
| 灵犀X2故障/不可用 | 实验无法进行 | 降级到桌面4轴臂+夹爪做简化版 |
