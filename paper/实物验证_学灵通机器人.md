# YLYW 实物验证：学灵通机器人运动控制

**版本：v1.0**  
**日期：2026年6月4日**

---

## 摘要

为验证YLYW架构从仿真到实物的跨平台迁移能力，将完整的L1→L2→L3→舵机推理链部署至基于STC15W4K8051单片机的学灵通人形机器人。仅依赖3轴陀螺仪作为传感器输入，YLYW引擎以纯定点整数运算在MCU上本地执行完整推理链，控制周期20ms（50Hz），无上位机依赖。实验验证了在极度受限的硬件条件下（8位MCU、无操作系统、仅3轴传感器），结构化先验知识仍能产生合理的实时步态决策。

---

## 1 实验平台

### 1.1 学灵通人形机器人

学灵通（XueLingTong, XLT）为一款教育/科研用人形机器人，关键硬件参数如下：

| 组件 | 规格 |
|------|------|
| 主控芯片 | STC15W4Kxx 系列 8051 兼容 MCU |
| 传感器 | 3轴陀螺仪（通过ADC读取） |
| 执行器 | 24路舵机（髋×4、膝×2、平衡×4、臂×8、踝等×6） |
| 通信接口 | UART1（串口，9600bps），支持外部状态注入或舵机角度直控 |
| 供电 | 锂电池包 |
| 自由度 | 12+ 主动自由度 |

### 1.2 传感器约束

与灵犀X2完整传感器套件（IMU、足端力传感器、深度相机、激光雷达）不同，学灵通仅配备**3轴陀螺仪**作为唯一的环境感知手段。这构成了对YLYW泛化能力的极限测试——在信息极度不完整的条件下，仅依赖结构化先验知识推断不可观测维度。

### 1.3 硬件约束

- **定点运算**：8051无浮点单元（FPU），所有运算必须转换为8/16/32位整数
- **内存限制**：片内RAM < 4KB，YLYW全部常量表（八卦原型48B + 六十四卦模板384B + 步态参数96B）存储在Flash中
- **实时性要求**：50Hz控制周期（20ms），推理+舵机计算需在周期内完成

## 2 总体架构

### 2.1 软件架构

系统由三个文件组成，完全运行于8051 MCU：

```
real_robot/
├── ylyw_8051.h          # YLYW推理引擎头文件（全部核心逻辑）
├── ylyw_standalone.c    # 独立运行模式（传感器→推理→舵机）
└── ylyw_uart_cmd.c      # 串口命令模式（PC→舵机角度直控，备选）
```

### 2.2 运行模式

**模式A（独立闭环）**：MCU内部完成传感器读取→YLYW推理→24舵机输出的完整闭环，无需上位机。由 `ylyw_standalone.c` 实现。

**模式B（串口控制）**：PC通过USB转TTL串口下发 `0xAA [24角度] 0x55` 帧，MCU直接转发至舵机，绕过推理引擎。基于 `ylyw_xlt_serial.py`（Python端）和 `ylyw_uart_cmd.c`（MCU端），用于调试、安全急停和对比实验。

## 3 传感器→6维状态映射

### 3.1 机制

`ylyw_read_sensors()` 从3轴陀螺仪的ADC原始值映射出6维归一化状态 `ylyw_state[0..5]`，每个维度为 `[0,255]` 的8位无符号整数。

**核心公式**：
```
dev[i] = value_present[i] - balance_value[i]   // 当前值 - 平衡参考值
tilt = |dev[pitch]| + |dev[roll]|              // 总倾角偏差
```

balance_value 为上电时记录的平衡参考值，确保姿态判断相对于初始校准姿态。

### 3.2 六维状态映射

| 爻位 | 名称 | 来源 | 映射逻辑 | 值域 |
|:--:|------|------|----------|:--:|
| 初爻 | 姿态稳定 | tilt（pitch+roll偏差） | 6档阶梯映射：<10→240, <20→200, <40→150, <80→100, <160→50, ≥160→20 | [20,240] |
| 二爻 | 质心高度 | tilt | 3档：<20→220, <60→170, ≥60→120 | [120,220] |
| 三爻 | 力分布 | 固定值 | **180**（无足底传感器，给中间值） | 180 |
| 四爻 | ZMP裕度 | tilt | 5档阶梯映射：<10→240, <30→180, <60→120, <120→60, ≥120→20 | [20,240] |
| 五爻 | 扰动 | Roll变化率 `|dev[roll]_t - dev[roll]_{t-1}|` | 5档：<5→230, <10→180, <20→120, <40→60, ≥40→20 | [20,230] |
| 上爻 | 地形 | 固定值 | **210**（无地形传感器，假定平坦） | 210 |

### 3.3 传感器约束下的状态空间降维

6维中仅**4维动态变化**（初爻、二爻、四爻、五爻），2维固定（三爻=180, 上爻=210）。与灵犀X2全传感器版的差异：

| 维度 | 灵犀X2（全传感器） | 学灵通（仅3轴陀螺） |
|------|------|------|
| posture | IMU → 俯仰/横滚角 | 陀螺 → tilt偏差 ✓ |
| com_height | 运动学正解 | tilt间接推断（质心≈f(倾角)）△ |
| force_dist | 足端力传感器 | **固定值** ✗ |
| zmp_margin | 足底压力中心 | tilt间接推断 △ |
| disturbance | IMU加速度+电机电流 | Roll角速率 ✓ |
| terrain | 深度相机 | **固定值** ✗ |

备注：✓=直接测量, △=间接推断, ✗=不可观测

尽管传感器信息严重不完整，YLYW的爻模板匹配对缺失维度具有天然鲁棒性：固定值180/210落在大多数爻模板的中间范围，不会极端地偏置匹配结果。

## 4 整数余弦相似度匹配

### 4.1 定点化公式

8051无FPU，余弦相似度必须转为整数运算。原始浮点公式：

$$\text{sim}(y, t) = \frac{y \cdot t}{\|y\| \cdot \|t\|}$$

定点化为：

$$\text{sim}_{\text{int}} = \frac{(y \cdot t / 10)^2}{(y \cdot y / 10) \cdot (t \cdot t / 10) + 1}$$

其中除法/10为近似归一化，避免32位乘法溢出。加1防止除零。平方操作等价于将线性相似度放大，增强Top-1匹配的区分度。

### 4.2 计算复杂度

| 操作 | 次数 |
|------|:--:|
| 64 × (6次8位乘法 + 累加) | 384次乘法 |
| 64 × (2次32位除法 + 1次乘法) | 128次除法 |
| 总计 | 约 512 次整数运算 |

在STC15W4K 8051 @11.0592MHz下，单次匹配约需1-2ms，远低于20ms控制周期。

## 5 六十四卦→步态→舵机映射

### 5.1 步态类型表

| ID | 步态 | 速度 | 步高 | 频率 | 力系数 |
|:--:|------|------|------|:--:|:--:|
| 0 | 静止站立 | 0 | 0 | 0 | 15 |
| 1 | 极慢爬行 | 5 | 15 | 4 | 25 |
| 2 | 慢走 | 12 | 30 | 6 | 45 |
| 3 | 正常行走 | 20 | 40 | 8 | 55 |
| 4 | 快速行走 | 30 | 50 | 10 | 70 |
| 5 | 小跑步态 | 45 | 60 | 13 | 65 |
| 6 | 奔跑 | 70 | 80 | 15 | 90 |
| 7 | 恢复步态 | 6 | 10 | 3 | 40 |
| 8 | 过渡缓冲 | 15 | 35 | 5 | 50 |
| 9 | 自适应步态 | 12 | 35 | 5 | 50 |
| 10 | 下坡步态 | 8 | 20 | 4 | 35 |
| 11 | 爬坡步态 | 10 | 50 | 5 | 50 |

参数值均为定点缩放量（非SI单位），直接适配舵机PWM范围。

### 5.2 正弦驱动

为回避8051上计算三角函数的高开销，采用**三角波近似正弦**：

```c
if (ph & 0x80) {
    sv = (s16)(256 - (ph & 0x7F)*2 - 1);   // 下降段: +127→-127
    cv = (s16)(-(ph & 0x7F)*2);            // 上升段: -127→+127
} else {
    sv = (s16)((ph & 0x7F)*2);             // 上升段
    cv = (s16)(256 - (ph & 0x7F)*2 - 1);    // 下降段
}
```

sv/cv分别用于左/右腿的正弦相位（相差π），实现交替摆腿。

### 5.3 舵机角度分配

| 舵机索引 | 功能 | 计算公式 |
|:--:|------|------|
| 0,2 | 右腿髋前后 | MID − lh |
| 3,5 | 左腿髋前后 | MID + lh |
| 1 | 右膝 | MID − lk（仅伸展相弯曲） |
| 4 | 左膝 | MID − lk（仅伸展相弯曲） |
| 8,11 | 左侧平衡 | MID ± bal |
| 9,10 | 右侧平衡 | MID ∓ bal |

其中 lh = ah × sv / 256, lk = ak × max(0, sv) / 256，均由步态参数的speed和force决定。

## 6 安全保护机制

### 6.1 平衡安全检查

`ylyw_balance_check()` 在每个控制周期对卦象输出做硬性覆盖：

```
if tilt > 60:  卦象 ← 24(地雷复), 步态 ← 恢复步态
elif tilt > 30: 卦象 ← 52(艮为山), 步态 ← 静止站立
```

这是基于纯物理约束的兜底机制——无论YLYW推理结果如何，当姿态倾角超过安全阈值时，强制切换到安全步态。注意30°阈值触发艮卦（静止如山），60°触发复卦（跌倒恢复），形成了两级渐进式安全响应。

### 6.2 与安全八卦的关系

该平衡检查可视为**嵌入安全机制的硬件实现**。在完整YLYW双八卦架构（论文第6节）中，安全八卦是一个独立的并行推理通道，输出SAFE/CAUTION/WARNING/DANGER/CRITICAL五级安全等级。学灵通上的平衡检查实现了安全协议的最底层子集——基于单一物理维度（倾角）的硬阈值保护。完整安全八卦的其余五个安全检查（破坏风险、摩擦锥、碰撞、稳定性裕度、穿透）在学灵通平台上因传感器缺失暂未实现。

## 7 主循环时序

```c
void ylyw_main(void) {
    while(1) {
        ylyw_step();         // 传感器→推理→舵机（完整一步）
        low_level_500u(40);  // 延时 40×0.5ms = 20ms
    }
}
```

单步详细分解：

```
ylyw_step():
  ├─ ylyw_read_sensors()     ADC采样+状态映射     ~0.5ms
  ├─ ylyw_infer()            64卦余弦匹配          ~1-2ms
  ├─ ylyw_balance_check()    倾角安全覆盖          ~0.1ms
  ├─ ylyw_servo_angles()     步态→24舵机角度      ~0.5ms
  └─ PWM_24()                舵机PWM输出           ~0.5ms
─────────────────────────────────────────────────────────
总计:                                            ~3ms
周期:                                            20ms
占用率:                                          ~15%
```

YLYW推理仅占15%的CPU时间，剩余85%可用于传感器滤波、通信、状态记录等。

## 9 学术贡献

本工作在以下维度对YLYW研究做出贡献：

1. **平台泛化验证**：将YLYW从灵犀X2仿真迁移到8051单片机实物，证明架构的硬件无关性。

2. **传感器贫瘠条件下的鲁棒性**：仅3轴陀螺仪（理论上无法观测力分布、ZMP、地形），YLYW通过结构化先验填补传感器空白，仍能产生合理的步态序列。

3. **极点部署**：在8位MCU（无FPU、无操作系统、RAM<4KB）上实现完整的L1→L2→L3→步态推理链，展示了结构化先验知识的极致计算效率——无需模型量化、无需模型压缩、无需神经网络推理框架。

4. **安全机制的嵌入式实现**：基于倾角的两级硬性安全兜底，验证了安全八卦设计哲学（物理约束作为符号先验）的实际可行性。
