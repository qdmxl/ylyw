# YLYW 运动控制验证实验方案（阶段三）

**版本：v1.0**  
**日期：2026年6月3日**  
**目标：从机器人运动控制角度验证YLYW模型的通用性与优势**

---

## 一、实验目标与学术价值

### 1.1 核心目标

验证YLYW作为**通用具身智能决策架构**的泛化能力——从静态物体交互（桌面抓取）扩展到动态身体控制（步态/平衡），证明结构化先验知识的有效性不局限于单一任务域。

### 1.2 学术价值

| 维度 | 说明 |
|------|------|
| **范式验证** | 从"一次性分类决策"扩展到"连续时变控制"，验证YLYW架构的通用性 |
| **稀缺生态位** | 运动控制是神经符号文献中**极其罕见**的应用场景——几乎空白 |
| **对比优势** | 运动控制是传统RL/MPC的强项，在此场景胜出更有说服力 |
| **叙事价值** | "用64卦控制机器人走路"是极具传播力的学术故事 |
| **实物闭环** | 灵犀X2提供从仿真到实物的完整验证链 |

### 1.3 与阶段一、二的差异

| 维度 | 阶段一/二（桌面抓取） | 阶段三（运动控制） |
|------|:--:|:--:|
| 状态空间 | 13维静态特征 | **连续时变 + 周期性** |
| 决策模式 | 一次性分类 | **实时连续输出** |
| 时间约束 | 无 | **毫秒级实时推理** |
| 物理约束 | 力上限校验 | **ZMP/摩擦锥/能量守恒 实时约束** |
| 失败代价 | 掉落物体 | **摔倒 → 灾难性** |
| 传统方法 | VLA/RL | **MPC/WBC/RL** |

### 2.2 六爻→状态变量映射

| 爻位 | 名称 | 物理语义 | 编码公式 | 灵犀X2传感器来源 |
|------|------|----------|----------|----------------|
| 初爻 | 躯干姿态 | 躯干倾角是否稳定（俯仰/横滚） | `0.5(1−|θ_pitch|/θ_max) + 0.5(1−|θ_roll|/θ_max)` | IMU → JointState |
| 二爻 | 质心高度 | 质心高度是否正常 | `1 − max(0, (h_nom − h_c)/h_nom)` | 运动学正解 + IMU |
| 三爻 | 足地接触力 | 各足端法向力是否均匀 | `1 − std(F_contact) / mean(F_contact)` | 足端力传感器 |
| 四爻 | ZMP裕度 | 零力矩点距支撑域边界的距离 | `min(d_zmp_to_boundary / d_safe, 1)` | 足底压力中心 |
| 五爻 | 扰动/能量 | 当前受外力扰动程度 + 能耗 | `0.5(1−|F_ext|/F_max) + 0.5(1−P/P_max)` | IMU加速度 + 电机电流 |
| 上爻 | 地形/环境 | 前方地形复杂度与障碍 | `1 − min(1, roughness × 2)` | 深度相机/激光雷达 |

所有变量归一化至[0,1]。≥0.5为阳爻（—），<0.5为阴爻（--）。

## 三、实验设计

### 3.1 总体路线

```
阶段 3.1 → 仿真验证（PyBullet + 灵犀X2 URDF）
    ├── 实验A: 步态模式分类（零样本）
    ├── 实验B: 抗扰动恢复
    ├── 实验C: 地形适应
    ├── 实验D: 消融实验
    └── 实验E: 对比实验（RL基线/FIS模糊控制）

阶段 3.2 → 实物验证（灵犀X2）
    ├── 实验F: 实机步态迁移
    └── 实验G: 实物扰动测试
```

### 3.2 仿真环境

- **物理引擎**: PyBullet 3.2.5+ 或 Isaac Gym（如可用）
- **机器人模型**: 灵犀X2 URDF（`lx2501_sdk`中提取）
- **地形生成**: 平地 + 随机障碍 + 斜坡（15°/25°/35°）
- **仿真步长**: 0.001s（1kHz物理），控制周期 0.02s（50Hz）
- **YLYW推理**: 每控制周期一次（50Hz），单次推理<1ms（远低于20ms周期）

### 3.3 实验A：步态模式分类（零样本基线）

**目的**：验证YLYW能否在零样本条件下正确识别当前步态模式并输出合理参数。

**配置**：
- 64卦步态模板库（预定义，基于易理语义 + 灵犀X2运动学参数）
- 11种步态模式 × 10种初始条件 → 110个测试场景
- 无任何训练数据

**评估指标**：
- 步态识别准确率（%）
- 速度参数合理性（目标速度 vs 物理可行速度的误差）
- 步高参数合理性（是否在关节限位内）
- 推理延迟（ms）

**预期结果**：零样本步态识别率 >85%，推理延迟 <2ms。

### 3.4 实验B：抗扰动恢复

**目的**：验证YLYW+L3+爻位关系能否在扰动下做出正确的安全响应。

**扰动类型**：
- 侧向推搡（100N/200N/300N，持续0.1s）
- 地面突然倾斜（5°/10°/15°斜坡弹出）
- 单腿打滑（降低足端摩擦至0.1）

**配置**：
- 每种扰动 × 3种步态（站立/行走/小跑）= 9种组合
- 各10次随机扰动时机 = 90个测试

**评估指标**：
- 恢复成功率（不摔倒回到稳定步态的比率）
- 跌倒率
- 爻位质量评分 S_yao 的预警能力（扰动前 vs 扰动后 S_yao 变化）
- 力修正系数的合理性（降力幅度是否与扰动强度正相关）

**预期结果**：恢复成功率 >80%，爻位质量在扰动发生50ms内显著下降（提前预警）。

### 3.5 实验C：地形适应

**目的**：验证YLYW对不同地形的零样本适应能力。

**地形类型**：
- 平地（基线）
- 上坡15°/25°/35°
- 下坡15°/25°
- 碎石地（随机0-3cm凸起）
- 台阶（5cm/10cm）

**配置**：每种地形 × 5次随机起点 = 30个测试

**评估指标**：
- 成功通过率（%）
- 平均速度保持率（实际速度/平地速度）
- 卦象切换频率（地形变化时是否及时切换到正确卦象）
- 最大躯干倾角（越小越稳定）

### 3.6 实验D：消融实验

沿用阶段二的消融方法论，验证YLYW各组件的独立贡献。

| 实验 | 条件 | 预期Δ | 验证假设 |
|------|------|:----:|----------|
| D1 | 易理 vs 随机打乱 | −30%+ | 步态卦象映射的有效性 |
| D2 | 完整L1+L2+L3 vs 仅L3 | −10%+ | 三层架构必要性 |
| D3 | 连续隶属度 vs 硬阈值 | −20%+ | 模糊表示优势 |
| D4 | 完整六爻 vs 仅3爻 | −15%+ | 六维状态空间的必要性 |
| D5 | 有爻位关系 vs 无爻位关系 | −10%+ | L3+层的扰动响应贡献 |

### 3.7 实验E：对比实验

| 方法 | 类型 | 训练数据需求 | 评估指标 |
|------|------|:----------:|----------|
| **YLYW（本工作）** | 知识驱动 | **0** | 步态识别率、恢复成功率 |
| PPO（RL基线） | 数据驱动 | 10⁷ 仿真步 | 同上 |
| 传统FIS模糊控制 | 专家规则 | 0（人工设计） | 同上 |
| MPC（模型预测控制） | 模型驱动 | 0（需精确模型） | 同上（仅平地） |

**关键对比点**：
- YLYW vs PPO：数据效率（0 vs 10⁷步）
- YLYW vs FIS：知识结构化程度（64卦网络 vs 平面规则）
- YLYW vs MPC：模型需求（无需精确模型 vs 需完整动力学）

## 四、代码架构

```
motion_control/
├── __init__.py
├── trigram_base.py          # L1: 八卦运动基元隶属度
├── yao_encoder.py           # L2: 六爻状态编码
├── hexagram_rules.py        # L3: 64卦步态模板库
├── yao_relations.py         # L3+: 爻位关系运算
├── safety_bagua.py          # 安全八卦（物理约束审核）
├── ylyw_locomotion.py       # 主控制器
├── simulation/
│   ├── lingxi_env.py        # PyBullet灵犀X2环境
│   ├── terrain_generator.py # 地形生成器
│   └── disturbance.py       # 扰动生成器
├── experiments/
│   ├── exp_a_zero_shot.py   # 实验A
│   ├── exp_b_disturbance.py # 实验B
│   ├── exp_c_terrain.py     # 实验C
│   ├── exp_d_ablation.py    # 实验D
│   └── exp_e_comparison.py  # 实验E
├── baselines/
│   ├── ppo_baseline.py      # RL基线
│   ├── fis_baseline.py      # 模糊控制基线
│   └── mpc_baseline.py      # MPC基线（可选）
└── real_robot/
    └── lingxi_adapter.py    # 灵犀X2实物接口适配
```

复用阶段一/二的 `trigram_base`、`yao_encoder`、`yao_relations` 核心模块，仅替换 `hexagram_rules`（从抓取策略→步态原型）和新增运动控制专用层。

## 五、灵犀X2实物验证

### 5.1 仿真→实物迁移策略

| 环节 | 仿真 | 实物 |
|------|------|------|
| 传感器 | PyBullet仿真传感器 | 灵犀X2 IMU/力传感器/深度相机 |
| 执行器 | PD控制器 | 灵犀X2 MC_control/JointCommand |
| 控制接口 | 直接调用 | 通过灵犀SDK ROS2接口 |
| 安全保护 | 无 | 安全八卦 + 硬件急停 + 人工监控 |

### 5.2 实物实验清单

| 实验 | 场景 | 时长 | 风险等级 |
|------|------|:--:|:--:|
| F1 | 平地行走（站立→慢走→站立） | 5min | 低 |
| F2 | 步态切换（慢走→小跑→慢走） | 3min | 低 |
| F3 | 轻度推搡恢复（100N侧推） | 2min | 中 |
| F4 | 不平地面行走（泡沫垫） | 3min | 中 |
| F5 | 上下斜坡（10°缓坡） | 3min | 中 |

### 5.3 安全协议

1. **软件层**：安全八卦实时审核所有输出指令
2. **硬件层**：急停按钮 + 安全绳
3. **人工层**：操作员手持遥控器，随时切换至手动模式
4. **渐进式**：先低速验证，逐步提升速度/扰动强度

## 八、风险与应对

| 风险 | 概率 | 影响 | 应对 |
|------|:--:|------|------|
| 灵犀X2 URDF与PyBullet不兼容 | 中 | 延迟 | 使用简化模型 + MuJoCo备选 |
