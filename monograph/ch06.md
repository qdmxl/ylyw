# 第6章 跨域实证：从感知到行动

第1-5章从哲学根基到架构设计到学习机制再到安全约束，完成了YLYW的完整理论体系。但理论体系的最后验证不是逻辑推导——而是**实验**。

本章报告YLYW在三个主要域（物理抓取、运动控制、具身导航）和三个补充域（视觉分类、书法轨迹、嵌入式部署）上的系统实验。这些实验的共同设计原则是：零样本优先、消融验证、对比评估、可解释评估。它们共同构成了对"结构化先验知识可以在多个域中替代大量训练数据"这一核心主张的跨域实证支撑。

---

## 6.1 实验设计的整体思路

### 6.1.1 为什么跨域

YLYW的架构论证中有一个根本性的主张：八卦隶属度→六爻编码→卦象匹配的三层推理架构是**域无关的**——它不绑定于特定的物理语义，而是一种通用的"感知→编码→决策"计算框架。跨域验证正是要检验这一主张。

如果YLYW仅在物理抓取域有效，可能是因为"64卦恰好适合描述物体属性"——但这个巧合的解释在逻辑上是不充分的。只有当相同的架构骨架（仅替换L1原型、L2公式、L3规则库的语义内容）在完全不同域（运动和交互、导航、感知）中同样有效时，才能证明架构的通用性。

### 6.1.2 四原则

所有跨域实验遵循同一套设计原则：

1. **零样本优先**：不调参、无训练数据的裸推，作为系统能力的"底线测试"。
2. **消融验证**：每个设计选择的独立贡献通过消融实验量化归因。
3. **对比评估**：与域内基准方法进行系统对比。
4. **可解释评估**：不仅报告"成功率"等宏观指标，还输出可追溯的推理链案例。

### 6.2.1 实验配置

第5.1节已详细描述了YCB测试集的评估方法论（双指夹爪力学模型、角度增益50°搜索、力闭合/提升/安全三维评分）。此处仅列出核心配置和结果。

**测试集**：50个物理物体（8大类：球体8个、立方体6个、碗6个、瓶子6个、圆柱体6个、盘子6个、石块6个、花瓶6个），来自YCB标准物体集和日常物品。每物体3次重复，共150次试验。每次叠加±5%随机噪声模拟传感器不确定性。

**零样本基线**：YLYW在纯零样本（无训练、无调参）条件下，使用完全初始的六十四卦规则推理。

## 6.3 运动控制域

### 6.3.1 迁移设计：从抓取到步态

三层架构的跨域迁移仅需重新定义每层的语义内容。物理抓取域使用物体物理属性（13维），运动控制域使用机器人状态（6维传感器）。

**L1八卦运动基元**。8个运动原型对应8种运动状态：乾（高速驱动）、坤（柔顺缓行）、震（动态扰频）、艮（静态稳定）、坎（恢复抗扰）、离（地形感知）、兑（协调节奏）、巽（自适应）。每个原型由6维向量定义。

**L2六爻状态编码**。6爻对应运动控制的六个核心维度：

| 爻位 | 物理语义 | 计算来源 |
|:----:|---------|---------|
| 初爻 | 躯干姿态稳定性 | IMU倾角(roll+pitch)→[0,1] |
| 二爻 | 质心高度 | 加速度计Z→低通滤波归一化 |
| 三爻 | 足端力分布均匀性 | 左右腿力矩比→[0,1] |
| 四爻 | ZMP稳定裕度 | 倾角+角速度组合评估 |
| 五爻 | 扰动/能耗 | 角速度幅度+垂向加速度变化 |
| 上爻 | 地形通过性 | 加速度高频分量/地面模型 |

**L3六十四卦步态策略库**。64卦映射为14种步态类型，涵盖从静止站立→极慢爬行→慢走→正常行走→快速行走→小跑→奔跑的完整速度谱系，以及谨慎行走、恢复步态、过渡缓冲、转向、爬坡、下坡、自适应等特殊步态。

### 6.3.3 MuJoCo G1人形机器人仿真

宇树G1人形机器人23-DOF URDF模型，MuJoCo物理引擎闭环仿真。200Hz控制频率，YLYW推理10Hz。

**推搡恢复实验**。15s时刻注入300N侧向推力：

```
14.0s 火风鼎(walk, 0.60) → 15.0s 坎为水(recovery, 0.20) → 
15.5s 地雷复(recovery, 0.20) → 16.5s 雷火丰(fast_walk, 0.75) → 
17.5s 泽雷随(walk, 0.55) → 18.0s 风泽中孚(walk, 0.55)
```

推搡瞬间姿态从0.65骤降至0.12，爻位关系检测到4处不当位和2处阴乘阳，力修正从0.60降至0.40。4秒后恢复至正常行走。

**爻位关系域间差异**。在运动控制域中，爻位关系修正系数均值0.94——显著低于静态视觉域的~1.0。验证了爻位关系设计用于处理时序动态变化的理论预期：六爻之间的"乘承比应"在动态协调中比在静态分类中更有实际效用。

## 6.4 ALFWorld具身导航域

### 6.4.1 迁移设计：从物理域到导航域

跨域迁移不仅改变了语义内容，还改变了任务性质——从"如何操作单个物体"变为"如何在多房间环境中导航并完成多步骤任务"。

**L1导航域八卦（8种位置原型）。** 5个标准位置(冰箱/微波炉/炉灶/水槽/桌子)映射为5个基本卦，3个通用探索原型(high_frequency/moderate_frequency/low_frequency)映射为3个动态卦。

**L2导航域六爻（6维任务状态编码）。** 温度状态(需要加热/冷却/常温)、清洗需求、可见状态、是否需要开门、容器类型、使用频率。

**L3导航域64卦策略模板。** 卦象匹配→具体行动序列，如[1,0,1,0,1,0]→"前往可见位置→取物→加热→放置"。

## 6.5 补充域

### 6.5.1 学灵通8051嵌入式验证

将YLYW推理链部署至STC15W4K8051单片机（8位MCU、4KB RAM、无FPU）。仅3轴陀螺仪作为唯一传感器，6维状态中2维固定、4维动态变化。整数余弦相似度简化后约512次运算，单次匹配1-2ms，控制周期20ms（占用率15%）。

7级步态测试序列（站立→慢走→行走→快走→小跑→奔跑→急停→恢复→站立）验证了在资源极度受限条件下YLYW架构仍能产生合理的实时步态决策。

### 6.5.2 书法轨迹学习

MuJoCo机械臂书写楷体汉字，10个测试字（大/人/永/中/山/水/天/地/国/心）。L1笔画空间分布→L2笔序/笔压/笔速/方向/曲率/对称性→L3结构卦→stroke序列。知几闭环迭代5-8轮收敛至可辨识。

### 6.5.3 视觉分类（STL-10）

STL-10零样本Top-1准确率37.0%（随机基线10.0%→3.7倍提升），Brodatz纹理数据集70.4%。性能受限于静态域中爻位关系修正≈1.0（不活跃）——与运动控制域形成鲜明对比，验证了爻位关系是为处理动态物理约束而设计的。
