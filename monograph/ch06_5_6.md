# 6.5 补充域与消融实验

第6.2-6.4节报告了YLYW在物理域（92.0% YCB）、运动控制域（11种步态零样本映射）和导航域（90.0% ALFWorld一致子集）上的系统实验。三个主要域的跨域验证已经充分支持YLYW架构域无关性的主张。

但YLYW的实验覆盖不止于此。本节报告四个额外的验证方向——嵌入式部署验证、书法轨迹学习、视觉分类验证、以及消融实验——它们分别从不同的角度支撑YLYW的核心主张。

---

## 6.5.1 嵌入式部署验证：学灵通8051

最严格的"域迁移"是**将YLYW架构部署到一个资源极度受限的硬件平台**。如果一套推理算法在8位MCU、4KB RAM、无FPU的条件下仍然能产生合理的实时决策，那么它的架构不依赖任何特定硬件假设的主张就得到了最极端的验证。

**平台约束。** 学灵通（XueLingTong）教育/科研人形机器人，主控芯片为STC15W4K系列8051兼容MCU，工作频率约35MHz，片内RAM < 4KB，无浮点运算单元（FPU），无操作系统。传感器仅3轴陀螺仪（通过ADC读取），执行器为24路舵机。YLYW引擎以纯定点整数运算在MCU上本地执行完整推理链，控制周期20ms（50Hz），无上位机依赖。

**定点化实现。** 将L1隶属度计算的浮点高斯核替换为整数查表：

```
// 定点余弦相似度: 6维向量 → [0, 1024] (定点10位)
int16_t dot = 0, norm_y = 0, norm_t = 0;
for (int i = 0; i < 6; i++) {
    dot += y[i] * t[6*h + i];
    norm_y += y[i] * y[i];
    norm_t += t[6*h + i] * t[6*h + i];
}
// sim = dot / sqrt(norm_y * norm_t) → 整数查表近似
```

全部常量表（八卦原型48B + 六十四卦模板384B + 步态参数96B）存储在Flash中，运行时仅使用片内RAM。单次卦象匹配约512次整数运算，耗时1-2ms，控制周期20ms（占用率约15%）。

**传感器信息贫瘠下的鲁棒性。** 6维状态中，2维完全不可观测（足端力分布、地形通过性），4维需要从仅有的3轴陀螺仪数据间接推断（躯干姿态=倾角积分、质心高度=Z轴低通滤波、ZMP裕度=倾角+角速度组合、扰动=角速度幅度）。YLYW在学灵通上的推理使用固定值填补不可观测维度，剩余4维动态变化。

**7级步态测试序列。** 完整的测试序列为：静止站立 → 慢走 → 行走 → 快走 → 小跑 → 奔跑 → 急停 → 恢复站立 → 静止站立。学灵通在全部7个步态阶段均产生了合理的YLYW推理输出和对应的舵机控制序列，没有出现推理链断裂或姿态控制崩溃。

**嵌入式验证的意义。** 学灵通8051实验验证了两个关键结论：
1. **YLYW的最小硬件需求接近零。** 一套可以在8051上运行（需定点化）的具身推理引擎意味着它可以被嵌入任何微控制器控制系统中——从智能家居设备到工业传感器节点，不需要GPU甚至不需要CPU。
2. **传感器贫瘠≠推理失败。** 6维状态中仅4维动态变化、2维固定填补的条件下，YLYW仍能产生合理的步态序列。因为先验知识（卦象→步态映射、爻位关系运算）提供了大量"预装"的结构信息，弥补了传感器数据的不足。

## 6.5.2 书法学习验证

第4章建立了知几学习机制——可在3轮内将物理抓取从58.0%收敛至66.0%。书法学习实验提供了一个不同模态的验证：通过知几闭环迭代，机械臂逐步改善汉字书写轨迹的质量。

**实验设置。** MuJoCo仿真中的机械臂腕部轨迹控制。10个测试汉字：大、人、永、中、山、水、天、地、国、心。输入为标准楷体字帖的图像骨骼点序列，输出为机械臂腕部运动轨迹。每次迭代后，将书写结果的骨骼点与标准字帖进行对比，计算偏差。偏差反馈经YLYW的知耻修正管道的回放，定位导致轨迹偏差的爻值参数，修改对应卦的模板值。

**三层映射。** L1八卦笔画空间分布（笔画的空间走向→八卦原型：乾=竖直以力贯之、坤=横向平顺、震=弹性抖动、艮=稳定止笔、离=轻快出锋、坎=含蓄回收、兑=圆转、巽=方折）。L2六爻编码（笔序、笔压、笔速、方向、曲率、对称性）。L3卦象匹配→结构卦→标准stroke序列。

**表6.22 书法学习知几闭环实验结果**

| 指标 | 数值 |
|-----|:----:|
| 测试字数 | 10 |
| 最大迭代次数 | 5 |
| 总迭代次数 | 19 |
| 平均初始偏差 | 0.610 |
| 平均最终偏差 | 0.580 |
| 平均改进幅度 | 0.030 (4.9%) |
| 收敛率（提前收敛至阈值以下） | 20.0% |
| 等级分布（差/中/良） | 11/6/2 |

5轮迭代内部分汉字的轨迹偏差持续减小（如"大"字在4轮后进入"中"等级），但整体收敛速度低于物理抓取。原因：书法轨迹的自由度（机械臂腕部6DOF）远高于抓取的力参数（1个标量值），单轮迭代的修正覆盖面有限。一个值得注意的发现是，部分汉字（如"永"）在前3轮中改进显著，随后进入振荡——因为笔序和笔压两个爻值之间存在耦合关系（加大笔压→需要放慢速度→笔序的连贯性下降），而YLYW的知耻修正机制当前未显式处理爻间耦合。

**书法≈焊接的同构论证。** 书法学习和焊接轨迹规划在数学上高度同构——两者均为"沿预设路径施加控制力的时序输出"。YLYW的易理轨迹规划思路（八卦编码物理约束：力、速度、倾角）可迁移至焊接、3D打印、数控加工等需要精确轨迹控制的工业场景中。

## 6.5.3 视觉分类域：STL-10零样本实验

视觉分类域提供了一个特殊的验证角度——它没有物理动态约束，没有时序变化，没有离散决策（如抓取策略类型或步态选择）。这正好检验了YLYW架构在非物理域中的泛化边界。

**实验设置。** STL-10图像数据集（10类，500训练/800测试/类，RGB 96×96）。Brodatz纹理数据集（未知类别数，灰度纹理图像）。不使用任何CNN或注意力机制，仅将图像的全局特征（8维视觉属性：**红/绿/蓝比例、亮度对比度、边缘密度、方向程度、纹理复杂度、对称性**）通过八卦隶属度映射为8维八卦向量，然后经L2→L3推理得到分类标签。

**表6.23 视觉分类零样本实验结果**

| 数据集 | YLYW零样本 | 随机基线 | 提升倍数 | 备注 |
|-------|:--------:|:--------:|:-------:|------|
| STL-10 Top-1 | 37.0% | 10.0% | **3.7×** | 零样本 |
| STL-10 Top-3 | 52.3% | 30.0% | **1.74×** | 前三名准确率 |
| Brodatz纹理 | 70.4% | — | — | 更高的纹理区分度 |

STL-10的37.0% Top-1准确率是随机基线（10.0%）的3.7倍。Brodatz纹理数据集上的70.4%说明YLYW的纹理特征编码方式在纹理分类上具有更好的判别力。但相比深度学习的SOTA（通常>90%），YLYW在纯视觉分类域上的表现仍然有限——这不是"YLYW不够好"的证据，而恰恰是验证了第3.4.5节关于爻位关系域相关性的论断。

**爻位关系在视觉域中失效。** 在STL-10实验中，爻位关系的力修正系数均值≈1.0，标准差<0.02。这意味着爻位关系运算在静态图像分类中几乎不产生任何效果——因为图像特征之间不存在"乘承比应"的物理动态关系。这验证了爻位关系是为处理物理交互的动态约束而设计的，在无物理动态的域中自然"自我静音"。
