# 8.4 与LLM智能体的对比

第8.2节对比了VLA（端到端映射），第8.3节对比了世界模型（环境预测→规划）。本节转向第三条主要路线：**LLM智能体**——以大语言模型为智能体"大脑"，通过提示工程、工具调用和代码生成与物理世界交互。

---

## 8.4.1 LLM智能体范式的本质

LLM智能体路线的核心范式可以概括为：**LLM（语言模型） + 工具调用 + 代码/动作生成 → 任务完成。** 它的基本前提是：大规模语言模型在预训练中捕获了大量的世界知识和推理能力，这些能力可以被"提示"从模型中提取出来，重构为对物理世界的操作指令。

**ReAct**（Yao et al., 2023）是这个范式的奠基工作。它的核心算法循环是：观察→思考→动作→观察→...。每一步，LLM接收自然语言观察，输出结构化"思考+动作"，动作通过预定义的API在环境中执行。ReAct在ALFWorld上的表现（~87%）证明了LLM的推理能力确实可以从语言任务迁移到具身导航任务。

**Code-as-Policies**（Liang et al., 2023）提出了一种更直接的交互方式：让LLM写Python代码来定义机器人行为。优势是利用LLM对代码语法的精确掌握来生成高度可控的动作序列，而不是让LLM在每一步进行"思考→动作"的在线推理。

**Voyager**（Wang et al., 2023）将LLM智能体推向了一个新高度——在Minecraft中，Voyager的LLM不仅能执行当前任务，还能自主编写新技能（用JavaScript代码表示），存储到技能库中，供未来任务调用。这展示了LLM智能体的元认知能力：它不仅能"做"，还能"学会如何做"。

---

## 8.4.2 LLM智能体的三个隐含假设及其系统性漏洞

LLM智能体路线隐含着三个核心假设，YLYW在每个假设上都提供了不同的视角。

**假设一：LLM的"世界知识"足够用于物理交互。** 这一假设的基础是LLM在海量互联网文本中见过的"物理世界描述"。GPT-4在预训练中读过数以百万计的"如何抓取物体"的文字描述——从维基百科文章到机器人论坛帖子。但这一假设的漏洞在于：**文本中的物理世界是被"叙述"的物理世界，不是被"仿真"的物理世界。** 模型读过的"球会滚动"是一个语言命题，不是一个物理直觉。当LLM读到"抓取力0.5是合适的"时，它从文本中学习的是一个数字0.5，而不是"为什么是0.5"的物理约束（物体的重量、摩擦系数、稳定性共同决定最优的0.5）。

这解释了为什么ReAct在ALFWorld中可以达到87%，但同样的LLM在纯粹的物理推理任务上（如判断"这个角度下物体会不会滑落"）表现远不如一个简单的物理公式。LLM知道"力=质量×加速度"——但它不知道将这个公式应用到任意新物体时的精确数值结果。YLYW的六爻编码通过公式聚合做到这一点——它不需要在文本中读过"锡皮球在油性表面上的抓取力应该是多少"。

**假设二：规划可以作为动作序列从LLM中直接生成。** Code-as-Policies的核心理念是："LLM写出Python代码→机器人执行代码"。这一假设的漏洞在于：代码生成时没有物理反馈——LLM不知道机器人执行第一步时实际发生了什么。它写的代码是一个预设序列，而不是一个闭环策略。

YLYW的推理是闭环的——每一步都根据当前的六爻状态确定策略。但YLYW在"高层次规划"方面确实弱于LLM智能体——YLYW不能从自然语言指令中分解出"先去厨房、再打开微波炉、再加热2分钟"这样的多步骤抽象计划。这是LLM智能体的核心优势——也是混合架构的机会。

**假设三：失败可以通过更多的"思考"来克服。** ReAct的失败恢复是通过在"思考"步骤中分析错误并重新规划——但这仍然是"语言层面的自我修正"，而非物理层面的诊断。如果失败的原因是"力不够大"（而非"选错了物体"），LLM没有渠道获得这一反馈——它的观察来自文本环境描述，而非力觉传感器。

## 8.4.3 LLM智能体 vs YLYW：五个核心维度

**表8.7 LLM智能体 vs YLYW**

| 维度 | LLM智能体（ReAct/Voyager/CoP） | YLYW |
|------|------|------|
| 知识来源 | 大规模互联网文本预训练 | 《易经》结构化先验 |
| 推理媒介 | 自然语言/代码 | 八卦→六爻→六十四卦 |
| 规划能力 | 强（多步骤、分层） | 弱（单步策略为主） |
| 物理理解 | 描述性（文本层面） | 因果性（公式聚合+关系运算） |
| 技能泛化 | 通过自动代码生成 | 通过域迁移（替换语义内容） |
| 数据效率 | 零样本（已有预训练） | 零样本（先验知识） |
| 安全保证 | 隐式（提示中的约束） | 显式（安全八卦） |
| 对LLM的依赖 | 强（大模型必需） | 零（独立推理） |
| 可解释性 | 思考步骤可读 | 推理链全透明 |
| 部署成本 | GPU或云端API | CPU/嵌入式 |

## 8.4.4 混合方案：LLM作"大脑" + YLYW作"直觉"

LLM智能体和YLYW存在天然的互补性。LLM擅长大尺度认知——理解自然语言、制定高层计划、编写代码、学习新概念。YLYW擅长小尺度物理——实时物理推理、安全约束、精确力控。

**分层混合架构。** 顶层（策略层）：LLM接收自然语言任务描述→分解为子任务序列→输出符号化的"意图"（{action: take, object: cup, location: desk}）。中层（翻译层）：YLYW将符号化意图翻译为物理参数——利用L1-L3推理链确定抓取策略、力参数和安全约束。底层（执行层）：机器人执行YLYW输出的物理参数，传感器反馈进入下一轮的六爻编码和分析。

这种分工类似于人类认知的双过程理论（Kahneman, 2011）：系统一（YLYW，直觉推理）——快速、自动、物理基础，系统二（LLM，逻辑推理）——慢速、有意识、抽象思维。两者分别处理不同形态的认知任务，合在一起覆盖完整的智能谱系。

第6.4节ALFWorld实验的结果为这一混合方案提供了实证启示：YLYW的物理推理在零样本条件下提供了正确的底层决策（"如何抓取这个物体"），而LLM在底层推理上的补充作用有限。但在高层规划（"完成这个多步骤任务需要哪些子任务"）上，LLM的贡献是YLYW单独难以替代的。
